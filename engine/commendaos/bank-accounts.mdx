## Overview

Use bank accounts to manage financial institution information for your business entities. Bank accounts store account details, routing information, and account types for corporations and other legal entities.

Bank accounts now support a country-aware `credentials` object that holds country-specific banking fields (such as IFSC codes for India or sort codes for the UK). You can retrieve the full set of supported fields per country from the [fields config endpoint](#get-bank-account-fields-config).

## Prerequisites

Before adding a bank account, ensure you have:

- A business entity created in CommendaOS
- The business entity ID
- Bank account details (account number, routing number, bank name)
- Account type information (checking, savings, etc.)

## Get bank account fields config

Retrieve the global fields configuration that defines which credential fields are required or optional for each country. Use this to render country-specific bank account forms.

```http
GET /bank-accounts/fields-config
```

**Response**

Returns a JSON object containing:

- `_meta` — metadata about the config version
- `field_definitions` — the full list of available fields with labels, types, and validation rules
- `countries` — per-country configuration listing `required_fields`, `optional_fields`, and any `field_overrides`

## Add a bank account

To add a new bank account to a corporation:

### Step 1: Gather required information

Collect the following details:

- **Bank name**: Full legal name of the financial institution
- **Account number**: Full bank account number
- **Routing number**: Bank routing/transit number (US) or equivalent
- **Account type**: Checking, savings, or other account type
- **Account holder name**: Legal name on the account (should match business entity)
- **Country**: The country where the bank account is held
- **Currency**: Account currency in [ISO 4217](https://en.wikipedia.org/wiki/ISO_4217) format (USD, EUR, INR, etc.)
- **Credentials**: Country-specific banking fields (see [fields config](#get-bank-account-fields-config))

### Step 2: Create the bank account

```http
POST /business-entity/:businessEntityId/bank-accounts
```

**Path parameters**

| Parameter | Type | Description |
|-----------|------|-------------|
| `businessEntityId` | integer | Unique identifier for the business entity |

**Request body**

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `bankName` | string | yes | Full legal name of the financial institution |
| `accountNumber` | string | no | Full bank account number |
| `routingNumber` | string | no | Bank routing/transit number |
| `accountType` | string | no | `CHECKING`, `SAVINGS`, `MONEY_MARKET`, or `ESCROW` |
| `accountHolderName` | string | no | Legal name on the account |
| `country` | string | no | Country code where the account is held |
| `currency` | string | no | ISO 4217 currency code |
| `credentials` | object | no | Country-specific credential fields (validated per country — see [fields config](#get-bank-account-fields-config)) |
| `isPrimary` | boolean | no | Whether this is the primary account |
| `bankAccountAdditionalInfo` | object | no | **Deprecated.** Use `credentials` instead |

```json
{
  "bankName": "First National Bank",
  "accountNumber": "123456789",
  "routingNumber": "021000021",
  "accountHolderName": "Acme Corporation",
  "country": "US",
  "currency": "USD",
  "credentials": {
    "accountType": "checking",
    "routingNumber": "021000021"
  },
  "isPrimary": true
}
```

<Note>
When `credentials.accountType` is set to `"checking"` or `"savings"`, the system automatically maps it to the corresponding account type and sub-type.
</Note>

**Response**

```json
{
  "id": "ba_abc123",
  "bankName": "First National Bank",
  "accountNumberMask": "****6789",
  "routingNumberMask": "021000021",
  "bankAccountType": "Checkings",
  "subType": "Checkings",
  "accountHolderName": "Acme Corporation",
  "country": "US",
  "currency": "USD",
  "credentials": {
    "accountType": "checking",
    "routingNumber": "021000021"
  },
  "additionalDetails": {},
  "address": null,
  "swiftCode": null,
  "statements": []
}
```

### Step 3: Verify the account

After creating the bank account, verify the details are correct:

```http
GET /business-entity/:businessEntityId/bank-accounts/:bankAccountId
```

**Path parameters**

| Parameter | Type | Description |
|-----------|------|-------------|
| `businessEntityId` | integer | Business entity ID |
| `bankAccountId` | string | Bank account ID |

## Update a bank account

To update existing bank account information:

### Step 1: Retrieve current details

First, get the current bank account information:

```http
GET /business-entity/:businessEntityId/bank-accounts/:bankAccountId
```

### Step 2: Update the account

```http
PATCH /business-entity/:businessEntityId/bank-accounts/:bankAccountId
```

**Path parameters**

| Parameter | Type | Description |
|-----------|------|-------------|
| `businessEntityId` | integer | Business entity ID |
| `bankAccountId` | string | Bank account ID |

**Request body**

Include only the fields you want to update:

| Field | Type | Description |
|-------|------|-------------|
| `bankName` | string | Full legal name of the financial institution |
| `currency` | string | ISO 4217 currency code |
| `credentials` | object | Country-specific credential fields. Merged with existing credentials so you can send partial updates without overwriting other fields |
| `bankAccountAdditionalInfo` | object | **Deprecated.** Use `credentials` instead |
| `isPrimary` | boolean | Whether this is the primary account |

```json
{
  "currency": "EUR",
  "credentials": {
    "iban": "DE89370400440532013000"
  }
}
```

**Response**

Returns the updated bank account object with `200 OK` status.

## Remove a bank account

To delete a bank account from a corporation:

```http
DELETE /business-entity/:businessEntityId/bank-accounts/:bankAccountId
```

**Path parameters**

| Parameter | Type | Description |
|-----------|------|-------------|
| `businessEntityId` | integer | Business entity ID |
| `bankAccountId` | string | Bank account ID |

**Response**

Returns `204 No Content` on success.

<Warning>
You cannot delete a bank account if managed filings is enabled for one or more of the entity's tax registrations. Disable managed filings first.
</Warning>

## Bank account response object

| Field | Type | Description |
|-------|------|-------------|
| `id` | string | Unique bank account identifier |
| `bankName` | string | Name of the financial institution |
| `accountNumberMask` | string | Masked account number (last 4 digits) |
| `routingNumberMask` | string | Masked routing number |
| `bankAccountType` | string | Account type (`Checkings`, `Savings`, etc.) |
| `subType` | string | Account sub-type |
| `accountHolderName` | string \| null | Legal name on the account |
| `country` | string \| null | Country code where the account is held |
| `currency` | string \| null | ISO 4217 currency code |
| `credentials` | object \| null | Country-specific credential fields |
| `additionalDetails` | object \| null | **Deprecated.** Legacy additional details. Read from `credentials` instead |
| `address` | object \| null | Address associated with the account |
| `swiftCode` | string \| null | SWIFT/BIC code |
| `statements` | array | Associated bank statements |
| `isRemittanceAccount` | boolean | Whether this is a remittance account |

## Best practices

- **Set a primary account**: Mark one account as primary for default transactions
- **Verify account numbers**: Double-check account and routing numbers before saving
- **Update regularly**: Keep bank account information current when accounts change
- **Secure storage**: Account numbers are automatically masked in responses for security
- **Currency matching**: Ensure the currency matches your business operations
- **Account holder names**: Use the exact legal name registered with the bank
- **Use credentials for country-specific fields**: Pass country-specific banking details (IFSC, IBAN, sort code, etc.) in the `credentials` object rather than the deprecated `bankAccountAdditionalInfo`
- **Use the fields config**: Call the [fields config endpoint](#get-bank-account-fields-config) to determine which credential fields are required for a given country

## Common account types

| Type | Description | Use case |
|------|-------------|----------|
| `CHECKING` | Standard checking account | Daily operations, payroll |
| `SAVINGS` | Savings account | Reserve funds |
| `MONEY_MARKET` | Money market account | Short-term investments |
| `ESCROW` | Escrow account | Held funds, transactions |
