---
title: "Update incorporation intake"
openapi: "POST /partner/incorporation/{incorporationId}/intake"
---

Updates partial or complete requirement data for an incorporation.

Each update is merged into the current intake state and validated against the current requirements DSL. Incomplete updates are accepted and return a structured validation summary.

The Partner API does not expose a submission history endpoint. To read progress after an update, call `GET /partner/incorporation/{incorporationId}` and inspect `intakeState`.

## Resource fields

Resource fields can use existing Commenda OS resources or inline data. When an existing resource is missing incorporation-specific details, include supplemental `data` on the same `existingResource` reference. Supplemental data is used for intake validation and does not mutate the Commenda OS resource during Phase 2.

```json
{
  "requirements": {
    "directors": [
      {
        "mode": "existingResource",
        "resourceId": "key_person_001"
      },
      {
        "mode": "inlineData",
        "data": {
          "firstName": "Asha",
          "lastName": "Raman",
          "email": "asha@example.com",
          "nationality": "SG",
          "passportNumber": "E1234567N",
          "dateOfBirth": "1988-02-14",
          "residentialAddress": {
            "mode": "existingResource",
            "resourceId": "location_001"
          }
        }
      },
      {
        "mode": "existingResource",
        "resourceId": "key_person_003",
        "data": {
          "passportNumber": "K9876543M",
          "residentialAddress": {
            "mode": "existingResource",
            "resourceId": "location_003",
            "data": {
              "postalCode": "079903"
            }
          }
        }
      }
    ]
  }
}
```

## Evidence linking

Upload files through Commenda OS first, link the existing `fileId` with `POST /partner/incorporation/{incorporationId}/files`, then reference that `fileId`:

```json
{
  "requirements": {
    "passportScans": [
      {
        "fileId": 456
      }
    ],
    "utilityBills": [
      {
        "fileId": 789
      }
    ]
  }
}
```

For Singapore private limited companies, both `passportScans` and `utilityBills` are required evidence requirements.
