---
title: "Update incorporation intake"
openapi: "POST /partner/incorporation/{incorporationId}/intake"
---

Updates partial or complete non-participant requirement data for an incorporation.

Each update is merged into the current intake state and validated against the current requirements DSL. Incomplete updates are accepted and return a structured validation summary.

The Partner API does not expose a submission history endpoint. To read progress after an update, call `GET /partner/incorporation/{incorporationId}` and inspect `intakeState`.

Participant roles and participant documents are managed through the participant and document endpoints, not by sending array indexes or document files inside this intake payload.

## Intake fields

Use this endpoint for incorporation-level requirements such as company name options and registered address selection.

```json
{
  "requirements": {
    "companyNameOptions": [
      { "name": "Acme SG Pte. Ltd." },
      { "name": "Acme Asia Pte. Ltd." },
      { "name": "Acme Global Pte. Ltd." }
    ],
    "registeredAddressMode": "standardAddress"
  }
}
```

If `registeredAddressMode` is `providedAddress`, include `providedRegisteredAddress` as a Commenda OS location reference.

```json
{
  "requirements": {
    "registeredAddressMode": "providedAddress",
    "providedRegisteredAddress": {
      "mode": "existingResource",
      "resourceId": "location_001"
    }
  }
}
```

## Singapore validation notes

For Singapore private limited companies, registered shareholder participant `ownershipPercentage` values must total 100.

If `registeredAddressMode` is `providedAddress`, include `providedRegisteredAddress`. If `registeredAddressMode` is `standardAddress`, no registered address resource is required.

For participant-specific requirements, register participants with roles and link documents to each participant:

- Use `POST /partner/incorporation/{incorporationId}/participants` for director/shareholder roles.
- Use `POST /partner/incorporation/{incorporationId}/documents` for documents such as `passport_scan` and `utility_bill`.

For Singapore private limited companies, each individual director and individual shareholder needs both `passport_scan` and `utility_bill` documents.
