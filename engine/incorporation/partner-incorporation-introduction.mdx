## Overview

The Partner Incorporation API lets a partner start and track incorporations for companies they manage in Commenda.

Base URL:

```text
https://api.prod.commenda.io/api/v1/partner/incorporation
```

Pass your partner API key in the `x-api-key` header on every request.

```bash
curl --request GET \
  --url 'https://api.prod.commenda.io/api/v1/partner/incorporation/jurisdiction-catalog' \
  --header 'x-api-key: <partner_api_key>'
```

## Flow

1. Create a company with the existing `POST /public/company` endpoint.
2. Review the informational jurisdiction catalog with `GET /partner/incorporation/jurisdiction-catalog`.
3. Fetch the current requirements DSL with `GET /partner/incorporation/requirements?country=SG&countryOptions[corporationType]=PRIVATE_LIMITED_COMPANY`.
4. Start an incorporation with `POST /partner/incorporation/companies/{companyId}/incorporations`.
5. Link already-uploaded files with `POST /partner/incorporation/{incorporationId}/files`.
6. Update partial or complete intake data with `POST /partner/incorporation/{incorporationId}/intake`.
7. Read the incorporation to track `intakeState`, missing requirements, and status.

This API does not create or return payment intents. Partners using this surface should treat billing as separate from the incorporation resource lifecycle.

## End-to-end Singapore example

This example assumes you already created a company and have a `companyId`. It starts a Singapore private limited company incorporation, links already-uploaded files, submits mixed intake data, and reads back progress.

```bash
curl --request GET \
  --url 'https://api.prod.commenda.io/api/v1/partner/incorporation/requirements?country=SG&countryOptions[corporationType]=PRIVATE_LIMITED_COMPANY' \
  --header 'x-api-key: <partner_api_key>'
```

```bash
curl --request POST \
  --url 'https://api.prod.commenda.io/api/v1/partner/incorporation/companies/<company_id>/incorporations' \
  --header 'content-type: application/json' \
  --header 'x-api-key: <partner_api_key>' \
  --data '{
    "country": "SG",
    "countryOptions": {
      "corporationType": "PRIVATE_LIMITED_COMPANY"
    }
  }'
```

## Jurisdiction options

Create requests use a `country` plus country-specific `countryOptions` instead of a flat product id.

```json
{
  "country": "US",
  "countryOptions": {
    "state": "DE",
    "corporationType": "LLC"
  }
}
```

See [List jurisdiction catalog](/engine/incorporation/partner-incorporation-jurisdiction-catalog-GET) for available country options and [Create an incorporation](/engine/incorporation/partner-incorporation-incorporations-POST) for request examples.

## Two-stage intake

The requirements endpoint returns a DSL with field names, descriptions, validation rules, resource fields, and evidence requirements. Use it to power your own UI, but treat it as informational: Commenda validates every intake update against the current server-side requirements.

Intake updates may be partial. Each update is merged into the current intake state and returns `intakeState.validation.isComplete: false` plus `missingRequirements` and `invalidRequirements` until the state is complete. Once the current intake state satisfies the DSL, the incorporation moves to `ready_for_review`.

## Evidence files

For files such as passport scans and utility bills, upload the file through Commenda OS first, then link that existing `fileId` through the incorporation file endpoint. The same `fileId` can be referenced from evidence fields:

```json
{
  "fileId": 456
}
```

Singapore private limited company intake currently requires both `passportScans` and `utilityBills`; one does not satisfy the other.
