The eSign feature allows you to send documents for electronic signature to customers and stakeholders directly from within a service request. Recipients receive email notifications and can sign documents using DocuSign's secure signing experience.

## How it works

<Steps>
  <Step title="Select or upload a document">
    From a service request, click the kebab menu (three dots) and select **Request Signature**. Upload a new PDF or select an existing document from the service request.
  </Step>
  <Step title="Select recipients">
    Add the people who need to sign — select from existing contacts or add new recipients by entering their name and email. You can add multiple recipients if multiple signatures are needed.
  </Step>
  <Step title="Place signature fields">
    Using DocuSign's embedded interface, visually place signature fields, date fields, text fields, and other form elements on the document. Assign fields to specific recipients.
  </Step>
  <Step title="Send the envelope">
    Review the document and click **Send**. DocuSign sends email notifications to all recipients.
  </Step>
</Steps>

## Signature tasks

When you send a document for signature, the system automatically creates signature tasks:

- **One task per recipient** — each person who needs to sign gets their own task
- **Visible in Tasks table** — tasks appear on the service page and Home screen

### Task statuses

| Status | Description |
|--------|-------------|
| **Pending** | Recipient is waiting to be notified, or waiting for a previous signer to complete (when using signing order) |
| **Sent** | Email has been sent to the recipient |
| **Viewed** | Recipient has opened the document |
| **Signed** | Recipient has completed signing |
| **Canceled** | Signature request has been canceled |

<Info>
If you configure a signing order (e.g., Signer 1 must sign before Signer 2), subsequent signers will remain in "Pending" status until the previous signer completes. Once the previous signer completes, the next signer's status updates to "Sent" and they receive their email notification.
</Info>

## Managing signature requests

### Sending reminders

You can send reminders to recipients who haven't signed yet by clicking **Send Reminder** on the signature task. Maximum of 1 reminder per day per recipient.

### Viewing signed documents

Once a recipient signs, the task status updates to "Signed" and agents can download the signed document from the task card. The signed document is stored with the service request.

### Canceling signature requests

Agents can cancel signature requests in progress. When canceled, all pending signatures are voided, recipients can no longer sign, and tasks are marked as "Canceled."

## Client experience

Recipients receive an email from DocuSign with a link to view and sign the document. No DocuSign account is required to sign. Clients also see signature tasks on their Commenda dashboard with an option to send a reminder.

## What happens after signing

1. The task status updates to "Signed"
2. The signed document becomes available for download
3. If multiple signers are required, the next recipient is notified
4. When all recipients sign, the envelope is marked as "Completed"

### When a recipient declines to sign

If a recipient declines, the entire envelope is automatically canceled — DocuSign voids all signatures. All tasks are marked as "Canceled" with a description of which recipient declined. A new signature request must be created.

## Notifications

All signature-related notifications are handled by DocuSign's native notification service (email reminders, confirmations, completion notifications). These are **not** sent through Commenda's notification system.

## Best practices

- **Clear document names** — use descriptive names so recipients know what they're signing
- **Multiple signers** — when multiple signatures are needed, consider the signing order
- **Timely reminders** — send reminders if recipients haven't signed within a reasonable timeframe
- **Document preparation** — ensure documents are final before sending for signature
- **Verify email addresses** — confirm recipient emails are correct before sending

## Troubleshooting

- **Recipient didn't receive the email** — check the email address, ask them to check spam/junk folders, or send a reminder
- **Need to make changes after sending** — cancel the current request, make changes, upload the revised document, and send a new request
- **Signing happens through DocuSign's interface** — recipients sign via the email link, not directly within Commenda
